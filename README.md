# Stock Price Trend Analysis — Spark SQL
**Problem Statement #19 | Big Data Analytics with Hadoop & Spark**

## Architecture

```
Yahoo Finance
     │  yfinance
     ▼
 [Producer] ──── Kafka (stock_prices topic) ──── [Spark Consumer]
                                                        │
                                              [PostgreSQL DB]
                                                        │
                                              [Streamlit Dashboard]

 [spark_analysis/] ← standalone Spark SQL demo (no Kafka/DB needed)
```

## Quick Start — Spark SQL Demo Only (Recommended for basic demo)

This runs the full Spark SQL analysis without needing Kafka or PostgreSQL.

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Copy env file
cp .env.example .env

# 3. Run Spark SQL analysis
python spark_analysis/spark_sql_analysis.py

# 4. Launch dashboard (reads output/full_analysis.csv)
streamlit run dashboard/app.py
```

The analysis fetches 180 days of OHLCV data for AAPL, MSFT, GOOGL, AMZN, TSLA
from Yahoo Finance and runs 8 Spark SQL queries:

| Query | Description |
|-------|-------------|
| Q1    | Summary statistics per ticker |
| Q2    | Moving averages (MA-7, MA-30) via Window functions |
| Q3    | Daily returns & 7-day rolling volatility |
| Q4    | Trend classification (UP / DOWN / NEUTRAL) |
| Q5    | Top 5 best & worst single-day returns |
| Q6    | Volume spike detection (> 2× average) |
| Q7    | Return vs. volatility comparison |
| Q8    | Save enriched dataset as **Parquet** (Hadoop-compatible) |

Output files are written to `output/`:
- `q1_summary_stats.csv`
- `q4_trend_distribution.csv`
- `q6_volume_spikes.csv`
- `q7_return_vs_volatility.csv`
- `full_analysis.csv`
- `stock_analysis_parquet/` ← partitioned Parquet (Hadoop format)

## Full Pipeline (Kafka + Spark + PostgreSQL)

### Prerequisites
- Docker Desktop
- Python 3.10+

### Steps

```bash
# 1. Start infrastructure
cd docker
docker-compose up -d

# 2. Install Python deps
pip install -r requirements.txt
cp .env.example .env

# 3. Start producer (fetches Yahoo Finance → Kafka)
python -m producer.main

# 4. Start consumer (Kafka → PostgreSQL via Spark or Python)
#    Option A – Python consumer (simpler):
python -m consumer.main

#    Option B – Spark Structured Streaming consumer:
spark-submit --packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.5.0 \
             consumer/consumer.py

# 5. Launch dashboard
streamlit run dashboard/app.py
```

### Services & Ports

| Service       | Port  |
|---------------|-------|
| Kafka         | 9092  |
| Zookeeper     | 2181  |
| PostgreSQL    | 5433  |
| Spark Master  | 8080  |
| Streamlit     | 8501  |
